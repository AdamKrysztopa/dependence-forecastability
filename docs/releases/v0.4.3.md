# v0.4.3

Release date: Planned

## Summary

v0.4.3 is a **Lag-Aware ModMRMR sparse covariate-lag selection and release-surface hardening** release.

It promotes a forecast-safe sparse covariate-lag selector into the stable core
surface via `run_lag_aware_mod_mrmr()`, with both a fast scorer mix and a
Catt-style KSG path, then stops at the framework-neutral
`ForecastPrepContract` hand-off. It remains a deterministic triage release.

It does **not** fit models, add downstream framework adapters, or move richer
walkthrough notebooks back into the core repo.

Plan: [docs/plan/implemented/v0_4_3_lag_aware_catt_mod_mrmr_plan_template_aligned.md](../plan/implemented/v0_4_3_lag_aware_catt_mod_mrmr_plan_template_aligned.md)

## Added

- `run_lag_aware_mod_mrmr(...)` as the public lag-aware sparse selector on the
  stable facade and `forecastability.triage`.
- Frozen typed input/output surface: `LagAwareModMRMRConfig`,
  `LagAwareModMRMRResult`, `ForecastSafeLagCandidate`,
  `SelectedLagAwareFeature`, `RejectedLagAwareFeature`,
  `BlockedLagAwareFeature`, `PairwiseScorerSpec`, `ScorerDiagnostics`, and the
  closed label aliases that describe known-future provenance, lag legality,
  normalization, significance, and rejection reasons.
- Canonical core showcase scripts
  `scripts/run_showcase_lag_aware_mod_mrmr.py` and
  `scripts/run_showcase_lag_aware_catt_mod_mrmr.py`, each with deterministic
  JSON, table, figure, and markdown outputs plus bounded `--smoke` mode.
- Dedicated theory and code-map pages for the selector surface in
  `docs/theory/lag_aware_mod_mrmr.md` and `docs/code/lag_aware_mod_mrmr.md`.

## Changed

- `README.md`, `docs/public_api.md`, `docs/examples_index.md`, and
  `docs/recipes/forecast_prep_to_external_frameworks.md` now document the
  lag-aware selector and the framework-neutral `ForecastPrepContract` hand-off
  without widening the public API to framework-specific helpers.
- `.github/workflows/smoke.yml` now exercises both lag-aware showcase scripts
  so the fast-scorer and Catt-style paths stay on the same smoke surface as the
  other deterministic showcases.
- `docs/releases/release_checklist.md`, `CHANGELOG.md`, and `RELEASES.md` now
  carry the lag-aware release-prep surface, including smoke parity and the
  lychee-offline CI-only note when local lychee runtime is unavailable.

## Notes

- mRMR and lag-aware / temporal mRMR-style selection are established. ModMRMR
  is a project-defined mRMR variant proposed by Adam Krysztopa. In this
  project the variant is the forecast-safe lag-domain plus multiplicative
  maximum-similarity suppression against already-selected features.
- The package remains framework-agnostic. No `darts`, `mlforecast`,
  `statsforecast`, or `nixtla` dependency or supported adapter surface is added
  here.
- Rich walkthrough notebooks remain a sibling-repo follow-up in
  `forecastability-examples`; this core release notes file does not claim that
  new notebooks were created in this repo.
