# Release v0.3.0 Upgrade Notes

Release date: 2026-04-18

## Summary

v0.3.0 promotes covariant-informative analysis to a first-class workflow while preserving
existing univariate forecastability interfaces.

## Highlights

- Adds stable covariant facade entry point via run_covariant_analysis.
- Introduces directional and dependence diagnostics in the covariant workflow (TE, GCMI)
  with optional causal adapters (PCMCI+, PCMCI-AMI).
- Adds covariant summary-table semantics and benchmark-backed examples/notebook walkthrough.
- Expands CI/release hardening: Python 3.11/3.12 matrix, smoke workflow, notebook contract
  coverage, release checklist template, and wheel smoke verification before publish.

## Known limitations

- v0.3.0 ships a finalized covariant-informative triage gate, not a full lagged-exogenous
  confirmatory causal toolkit.
- For strict lagged-exogenous causality confirmation, use the optional causal path and refer to
  the v0.3.1 follow-up plan.

## Upgrade notes

- No breaking import changes for existing univariate integrations.
- Existing run_triage and ForecastabilityAnalyzer usage remains valid.
- Covariant adoption is incremental through run_covariant_analysis.

## Compatibility

- Supported Python versions: 3.11 and 3.12.
- Optional extras: causal, transport, agent.

## Artifacts

When tag v0.3.0 is pushed, release workflows publish:

- Source distribution (sdist)
- Wheel distribution (wheel)
- GitHub Release artifacts from the same validated dist bundle
