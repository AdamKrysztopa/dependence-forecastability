# v0.3.6

Release date: 2026-04-25

## Summary

v0.3.6 is a stabilization release focused on release-truth and documentation-integrity automation for deterministic forecastability triage surfaces. It does not introduce runtime API-breaking changes.

## Added

- `scripts/check_repo_contract.py` release-truth checker for package version, plan-header coherence, release-notes presence, deprecated-path usage, and dependency-group policy.
- `scripts/sync_repo_contract.py --write` deterministic autofix path for configured repository-contract rewrites.
- `scripts/check_markdown_links.py` internal markdown-link integrity checker for `README.md`, `docs/**/*.md`, and `llms.txt`.
- `scripts/check_readme_surface.py` README landing-surface checker enforcing notebook references as supplementary/transitional.
- `scripts/check_published_release.py` post-publish verification for PyPI version visibility and GitHub release-tag presence.
- `repo_contract.yaml` as the single machine-readable configuration surface for release-truth and docs-integrity automation.
- `.github/workflows/repo-autofix.yml` workflow-dispatch/weekly automation to run repository sync and open a PR when rewrites are produced.

## Changed

- `.github/workflows/ci.yml` now includes a `repo-contract` job to run repository-contract and docs-integrity checks in PR/push CI.
- `.github/workflows/release.yml` now performs tag/version/release-notes guard checks and release-tag contract verification before build.
- `.github/workflows/publish-pypi.yml` now includes a post-publish verification job invoking `scripts/check_published_release.py`.
- Root `README.md` now uses supplementary/transitional notebook wording and removes forbidden notebook-path heading surface.
- Dependency group naming updated from `notebook` to `examples` in `pyproject.toml`.
- Regression fixtures and tests expanded under `tests/fixtures/repo_contract/` and `tests/test_repo_contract.py`, including end-to-end sync/checker smoke coverage.

## Notes

- The core package remains framework-agnostic.
- pAMI remains a project extension while AMI remains the paper-aligned foundation.
