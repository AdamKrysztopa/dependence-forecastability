# v0.4.1

Release date: 2026-05-04

## Summary

v0.4.1 is a **performance hardening, benchmark visibility, and fast-screening controls** release.

It makes performance a measurable, controllable surface: ships measurement infrastructure,
correctness guards for significance services, work-avoidance for explicit method subsets,
validation/scaling hygiene, a reduced eager import surface, and user-visible significance controls.

It does **not** eliminate the dominant surrogate/significance bottlenecks — those are identified,
scoped, and deferred to 0.4.2+. PCMCI and PCMCI-AMI are explicitly defocused as confirmatory
tools due to prohibitive runtime in practice.

Plan: [docs/plan/implemented/v0_4_1_performance_bottleneck_elimination_ultimate_plan.md](../plan/implemented/v0_4_1_performance_bottleneck_elimination_ultimate_plan.md)

## Added

- `n_jobs_significance: int = 1` parameter on `run_covariant_analysis` — passes through to
  `compute_significance_bands_generic` so callers can opt into parallel surrogate evaluation
  without changing the default serial behaviour.
- `yfinance` optional extra `[data]` — `yfinance>=0.2` moved out of core dependencies into
  `pip install "dependence-forecastability[data]"`. `scripts/download_data.py` emits a clear
  error when the extra is not installed.
- PCMCI/PCMCI-AMI defocus notes in `docs/quickstart.md` and `docs/public_api.md` — explicitly
  documents the confirmatory role, expense, and `parcorr` fast-path recommendation.
  Prefer `methods=["cross_ami", "gcmi"]` for routine triage.
- `configs/performance_baseline.yaml` — deterministic small/medium/large benchmark cases with
  explicit wall-time budgets.
- `scripts/run_performance_baseline.py` — repeatable timing script writing
  `outputs/performance/performance_summary.json` with full provenance (platform, git SHA,
  config hash, BLAS tag, CPU model, threadpoolctl snapshot, installed version).
- `scripts/profile_hotspots.py` — cProfile target registry writing
  `outputs/performance/hotspots.csv` and `.prof`/`.pstats` artifacts.
- `forecastability.ports.kernels` — typed `KernelProvider` Protocols for optional native
  acceleration (out-of-tree Rust/PyO3 plugin path; pure-Python remains default).
- Benchmark coverage expanded to covariant, lagged-exog, and import-latency workflows
  (PBE-F20, F21).

## Changed

- `n_surrogates` floor check in `run_covariant_analysis` and `run_lagged_exogenous_triage`
  now only enforces ≥ 99 when `significance_mode="phase"`; `significance_mode="none"` silently
  skips the check, making fast screening feel intentional.
- `src/forecastability/__init__.py` reduced eager import surface — only the canonical core
  symbols load at import time (`run_triage`, `run_covariant_analysis`,
  `run_lagged_exogenous_triage`, `build_forecast_prep_contract`, `TriageRequest`, `TriageResult`,
  primary result types, `generate_ar1`, `generate_white_noise`). All other `__all__` symbols
  resolve lazily via `__getattr__`.
- Covariant method-subset work avoidance (PBE-F03) — `methods=["cross_ami"]` no longer
  computes pCrossAMI; `methods=["cross_pami"]` no longer computes CrossAMI significance bands.
- Surrogate result matrices preallocated; list/`vstack` replaced (PBE-F05 — wall-time
  budgets not met; shipped as hygiene).
- Generic curve validation/scaling hoisted out of per-surrogate and per-lag loops (PBE-F16).
- Single-horizon rolling-origin path (PBE-F04) — rolling-origin no longer computes full
  curves when only one horizon is consumed.
- pAMI residualization replaced with closed-form `lstsq` (PBE-F06).
- Lagged-exog `n_jobs` pass-through and distance-correlation budget guard (PBE-F08).
- Fingerprint geometry feasibility cap and script `n_jobs` presets (PBE-F07).
- Batch workbench `n_jobs` with deterministic ordering (PBE-F12).
- PCMCI-AMI `n_permutations`, `ci_test`, per-method `max_lag`, `verbosity`, and Phase 0
  joblib parallelism exposed (PBE-F24, F25, F27).
- Release title renamed from "Performance Bottleneck Elimination" to "Performance hardening,
  benchmark visibility, and fast-screening controls" to accurately reflect delivered scope.
- Plan `**Current released version:**` and branch reference updated to match shipped state.

## Fixed

- `compute_significance_bands_generic` now validates `n_surrogates >= 99`, `which in {"raw",
  "partial"}`, `max_lag`, `min_pairs`, `n_jobs`, and input shapes before allocating surrogate
  matrices (PBE-F00 correctness gate).
- Fixed-seed regression test pins legacy `compute_significance_bands` band arrays so
  downstream optimization PRs cannot silently drift significance outputs.

## Deferred to 0.4.2+

- Batched kNN MI / exact KSG-II native kernel implementation
- Rust/maturin plugin
- Common-surrogate stream redesign
- Deeper PCMCI acceleration
- `matplotlib` behind `[plotting]` optional extra
