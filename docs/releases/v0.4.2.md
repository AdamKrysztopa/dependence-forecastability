# v0.4.2

Release date: 2026-05-04

## Summary

v0.4.2 is an **AMI-first forecastability structure expansion** release.

It extends the univariate forecastability fingerprint with additive spectral,
ordinal, classical, and memory diagnostics, plus an extended
forecastability-profile routing surface. The new layer is explanation-oriented:
it helps describe why a series may be forecastable before downstream model
search.

It does **not** fit models, add framework-specific training helpers, or expand
the causal-discovery surface. AMI geometry remains the primary lagged-
information evidence.

Plan: [docs/plan/v0_4_2_forecastability_structure_expansion_ultimate_plan.md](../plan/v0_4_2_forecastability_structure_expansion_ultimate_plan.md)

## Added

- `run_extended_forecastability_analysis(...)` as the direct public univariate
  entry point for the extended fingerprint.
- `ExtendedForecastabilityAnalysisResult`,
  `ExtendedForecastabilityFingerprint`, and
  `ExtendedForecastabilityProfile` on the stable public facade and
  `forecastability.triage`.
- Spectral, ordinal, classical, and memory diagnostic blocks that stay
  additive around the existing AMI/pAMI foundation.
- `forecastability extended` CLI surface with `json`, `markdown`, and `brief`
  output modes.
- Theory and usage docs for spectral forecastability, ordinal complexity,
  classical structure features, memory diagnostics, and the extended profile.

## Changed

- `run_triage(..., include_extended_fingerprint=True)` can now attach
  `extended_forecastability_analysis` for non-exogenous requests without
  changing the default triage contract.
- Deterministic routing now exposes `predictability_sources`,
  `recommended_model_families`, `avoid_model_families`, `signal_strength`, and
  `noise_risk` on `ExtendedForecastabilityProfile`.
- When AMI geometry is disabled or unavailable, the extended router switches to
  descriptive-only mode instead of emitting routing-grade family
  recommendations from secondary diagnostics alone.

## Notes

- The extended surface is **AMI-first**. Spectral, ordinal, classical, and
  memory diagnostics explain possible structure sources; they do not replace
  lagged-information analysis.
- No downstream model fitting, forecasting-framework integration, or causal
  expansion ships in this release.
- Rich walkthroughs and notebooks for this surface remain in the sibling
  `forecastability-examples` repository.
